# 感謝フォトアルバム Studio

会社の本部長の異動に際して、写真と言葉で感謝を伝えるための本番用フォトアルバム制作アプリです。

写真をただ並べるのではなく、歩み、功績、人柄、現場に残してくださったもの、感謝、未来へのエールを、ひとつの物語として構成します。

## 目的

- 写真を大切に扱い、感謝が伝わる送別アルバムを作る
- 職場で使って失礼がない、上品で落ち着いた上映物にする
- 本番上映とPDF保存までを、会社PCで扱いやすい形にする
- ChatGPTを使う場合も、写真そのものは送らず、プロンプト生成方式で文章支援を受ける

## 使い方

1. GitHub Pagesの公開URLをブラウザで開きます。
2. 「基本情報入力」でタイトル、贈る相手、日付、作成者、目的、トーンを入力します。
3. 「新しいアルバムを作成する」から「Golden Legacy 10枚パッケージ」「空のアルバムから作成」「自動テーマ構成で作成」を選びます。
4. スターターパッケージを選ぶと、表紙からエンディングまでの10枚構成が作られます。
5. スライド種別、写真枚数、フレーム配置テンプレート、感情役割を必要に応じて調整します。
6. 「このスライドに写真をまとめて追加」または各フレームの「写真追加」から、そのスライドに必要な写真だけを入れます。
7. スライドのタイトル、メッセージ、キャプションを整えます。
8. 必要に応じて「写真ライブラリ」で候補写真を事前にまとめて追加し、未使用写真から選びます。
9. 「AI支援画面」でChatGPT相談用プロンプトを生成し、必要に応じて文章案を貼り戻します。
10. 「上映プレビュー」でスライドを確認し、フルスクリーン上映、印刷/PDF保存、PPTX保存を行います。
11. 保存するときは、普段の編集用には「軽量JSON保存」、写真データも含めて予備を残す場合は「画像込みバックアップ保存」を使います。

## 制作モード

このアプリには3つの開始方法があります。

### スターターパッケージから作成

この機能を使うと、最初から10枚前後のスライド構成が生成されます。
写真・言葉・レイアウトは後から自由に編集できます。
ゼロから作るのが難しい場合や、送別アルバムの流れを先に作りたい場合に便利です。

Golden Legacy 10枚パッケージでは、表紙、導入、歩み、現場での姿、ご指導いただいた時間、みんなとの時間、残してくださったもの、感謝、未来へのエール、エンディングの10枚を作成します。各スライドにはDRM型の感情役割、推奨レイアウト、写真枚数、フレーム配置、仮タイトル、仮メッセージ、写真プレースホルダーが入ります。

### 自動テーマ構成モード

写真をまとめて取り込み、テーマごとに自動レイアウトする従来の補助モードです。写真候補が多く、まず全体の流れを機械的に作りたい場合に使います。

### スライド手動構成モード

メインの制作モードです。スライドを1枚ずつ作成し、スライド種別、写真枚数、フレーム配置テンプレート、感情役割を先に決めてから、そのスライドに必要な写真だけを追加します。本番用の贈り物として「このスライドは写真1枚を大きく見せる」「このスライドは3枚で流れを見せる」といった判断を反映できます。

## 写真の扱い

このアプリは写真を外部に送信しません。写真はお使いのブラウザ内で処理されます。

このアプリはGitHub Pages上で動作しますが、本番写真をGitHubにアップロードする必要はありません。公開するのは `index.html`、`css`、`js`、`README.md` などのアプリ本体だけです。写真は会社PCからブラウザで選択し、ブラウザ内でCanvasリサイズされます。

- 画像処理はブラウザのCanvasで行います。
- 表示用画像は最大幅約1600pxに縮小します。
- サムネイル画像は最大幅約320pxに縮小します。
- OpenAI APIキーは使いません。
- フロントエンドJavaScriptにAPIキーを書く必要はありません。
- localStorageに画像データは保存しません。
- 保存はプロジェクトJSON書き出しとJSON読み込みを優先します。
- 必要に応じてブラウザ内のIndexedDBにも保存します。
- 軽量JSON保存は画像データを含みません。
- 画像込みバックアップ保存は画像データを含むため、写真枚数が多い場合はファイルサイズが大きくなります。

## ChatGPT連携

ChatGPT連携はAPI接続ではありません。

アプリが生成するのは、ログイン済みChatGPTに手動で貼るためのプロンプトです。プロンプトには、テーマ情報、写真ファイル名、写真メモ、キャプション案、文章下書きなどが含まれます。写真そのものを自動送信する機能はありません。

ChatGPTから返ってきた文章は、ユーザーがアプリ内の貼り戻し欄に手動で貼り付けて使います。

## 対応画像形式

- jpg
- jpeg
- png
- webp

HEICはブラウザ対応が不安定なため、事前にJPGへ変換してから使用してください。

## スライドごとの言葉編集

「スライド編集」画面では、自動生成されたスライド文を1枚ずつ確認し、必要なスライドだけ手動編集できます。

スライド手動構成モードでは、左側にスライド一覧、中央に選択中スライドのプレビュー、右側に写真枚数・フレーム配置・写真割り当て・文字編集を表示します。

編集できる項目は以下です。

- スライドタイトル
- サブタイトル
- メインメッセージ
- 写真キャプション
- 補足メッセージ
- エンディングコピー
- レイアウトタイプ
- 感情役割

手動構成モードでは、以下も編集できます。

- スライド種別
- 写真枚数
- フレーム配置テンプレート
- 各フレームの写真
- 写真の表示方法
- 写真の位置
- 拡大縮小
- 回転
- フレーム内の写真入れ替え

写真は、スライド編集中に追加する方式が基本です。「このスライドに写真をまとめて追加」を使うと空いているフレームへ順番に仮配置され、各フレームの「写真追加」「写真差し替え」「写真削除」で個別に調整できます。事前にまとめて取り込んだ写真は「写真ライブラリ」として補助的に使えます。どちらの場合も画像は外部送信されず、ブラウザ内でCanvasリサイズされます。

レイアウト選択時には、写真選びのガイドを表示します。

- Hero Layout: 主役として見せたい写真を1枚選びます。
- Split Layout: 対比や組み合わせを見せたい2枚を選びます。
- Story Layout: 流れが伝わる3枚を選びます。
- Mosaic Layout: 同じテーマに属する写真を複数選びます。
- Message Layout: 写真なし、または少なめで言葉を主役にします。

写真枚数ごとに、以下のようなフレーム配置テンプレートを選べます。

- 0枚: 中央メッセージ、左寄せメッセージ、大見出し＋短文、余白多めの映画的レイアウト
- 1枚: 全面写真＋下部キャプション、Text Top / Image Bottom、Golden Legacy Text First、中央写真＋上下余白、左に写真、右にメッセージ、背景写真＋言葉、Golden Legacy風額縁
- 2枚: 左右2分割、Text Top / Two Images Bottom、上下2分割、メイン写真＋サブ写真、Before / After風、対比レイアウト
- 3枚: 大1枚＋小2枚、Text Top / Gallery Bottom、横並び3枚、縦並び3枚、三角配置、ストーリー順配置
- 4枚: 2×2グリッド、Text Top / 4 Image Gallery、大1枚＋小3枚、横長1枚＋下に3枚、左にメッセージ右に4枚、Golden Legacy風フレーム4枚
- 5〜8枚: コラージュ、モザイク、Text Top / Mosaic Bottom、ギャラリー、メイン写真＋小さな写真群、思い出の断片レイアウト
- 9枚以上: サムネイルギャラリー、思い出ウォール、ランダム風コラージュ、複数スライド化の提案

自動生成時の文章は「自動生成文」として保持されます。ユーザーが入力した文章は「手動編集文」として保存され、手動編集文がある場合はプレビュー、PDF保存、構成表出力で手動編集文を優先します。手動編集文が空欄の場合は、自動生成文を表示します。

## 言葉の重複チェック

「言葉の重複チェック」では、同じタイトルやキャプションの連続、「ありがとうございました」の連続使用、「思い出」「感謝」「支え」「新天地」などの偏り、似たメッセージ、文章量のばらつき、同じ感情役割の連続を確認します。

チェック結果は「問題あり」「注意」「問題なし」で表示します。結果をもとに、ChatGPTへ貼るための「言い換えプロンプト」も生成できます。

## PDF保存方法

1. 「上映プレビュー」を開きます。
2. 「PDF出力モード」を選びます。
3. 「印刷 / PDF保存」を押します。
4. 印刷画面で保存先を「PDFに保存」にします。
5. 操作パネルは印刷されず、1スライド1ページで出力されます。

PDF出力モードは以下から選べます。

- 16:9スライドPDF 推奨: 上映、PowerPoint代わり、デジタル共有向けです。PDFページ自体を16:9にして、スライドがページ全体に収まるようにします。
- A4横・中央配置: 紙にA4横で印刷する場合に使います。16:9スライドを比率維持で中央に置くため、上下に余白は出ますが下だけに偏りにくくなります。
- A4横・余白少なめ トリミングあり: A4横で余白を目立たせたくない場合に使います。黒背景でページ全体を覆い、スライドを拡大するため一部が切れる可能性があります。

A4横用紙と16:9スライドは比率が異なるため、A4印刷では上下に余白が出る場合があります。上映やデジタル共有には、16:9スライドPDF または PPTX出力を推奨します。

編集画面の中央プレビュー、上映プレビュー、PDF保存は共通のスライドレンダリングを使います。会社PCの印刷画面では、余白なし、背景グラフィックあり、選択したPDF出力モードに合う用紙設定にするとプレビューに近くなります。

PDFとPPTXの使い分け:

- 上映する場合: PPTX または 16:9スライドPDF
- 紙に印刷する場合: A4横・中央配置PDF
- 余白を減らしたい場合: A4横・余白少なめモード。ただし一部トリミングの可能性があります。

A4用レイアウト変換モードは将来対応予定です。これは16:9スライドをそのまま置くのではなく、A4横比率に合わせて文字領域や写真フレームを再計算する大きな改修になります。

## PPTX保存方法

1. 「上映プレビュー」を開きます。
2. 「PPTXとして保存」を押します。
3. `gratitude-photo-album.pptx` が保存されます。

PPTX出力は、見た目の崩れを防ぐため、各スライドを画像としてPowerPointに貼り付ける方式です。PowerPoint上で文字や写真を個別編集することはできません。

PPTX生成はブラウザ内で行い、外部CDNには依存しません。写真枚数が多い場合や会社PCの性能によっては、保存まで少し時間がかかります。

## プロジェクト保存方法

保存方式は2種類あります。

### 軽量JSON保存

画像データを含めず、基本情報、テーマ情報、写真順、写真ファイル名、写真メモ、キャプション、レイアウト設定、感謝メッセージ、AIから貼り戻した文章を保存します。

スライドごとの手動編集文、感情役割、言葉の調整内容も保存されます。
スライド手動構成モードのスライド種別、写真枚数、フレーム配置、フレームごとの写真割り当て、表示位置、拡大率、回転、表示方法も保存されます。

ファイルサイズが小さいため、日々の編集途中の保存や構成確認に向いています。ただし、画像データは含まれないため、読み込んだ直後は写真そのものは復元されません。上映やPDF保存を行う場合は、画像込みバックアップを使うか、写真を再度取り込んでください。

### 画像込みバックアップ保存

表示用画像とサムネイル画像を含めて保存します。別PCで開く場合や、本番前の予備ファイルとして残す場合に向いています。

写真枚数が多い場合はJSONファイルが大きくなります。50枚以上の写真がある場合、保存前に注意メッセージを表示します。

## 本番上映前チェックリスト

- 写真の順番は自然か
- 名前と役職に誤りはないか
- 日付は正しいか
- キャプションに誤字脱字はないか
- 軽すぎる表現がないか
- 内輪ネタが強すぎないか
- 最後の感謝メッセージは失礼がないか
- 写真に問題のある情報が写っていないか
- フルスクリーンで再生できるか
- PDF保存したか
- 本番用PCで動作確認したか
- 予備ファイルをUSB等に保存したか

## GitHub Pages公開方法

会社PCにPythonなどのローカルサーバーが入っていなくても、GitHub Pagesの公開URLを開けば使用できます。

### 1. GitHubリポジトリを作成する

1. GitHubにログインします。
2. 右上の `+` から `New repository` を選びます。
3. リポジトリ名を入力します。例: `gratitude-photo-album-studio`
4. 公開範囲を選びます。社内運用ルールに合わせて `Private` または `Public` を選択してください。
5. `Create repository` を押します。

### 2. アプリ本体だけをアップロードする

リポジトリのルートに、以下をそのままアップロードします。

```text
/index.html
/css/style.css
/js/app.js
/js/imageProcessor.js
/js/storage.js
/js/slides.js
/js/layoutEngine.js
/js/slideRenderer.js
/js/aiPromptBuilder.js
/js/exporter.js
/README.md
```

本番写真はGitHubにアップロードしません。写真フォルダ、画像ファイル、完成アルバム用の写真素材はリポジトリに入れないでください。

### 3. GitHub Pagesを有効にする

1. GitHubのリポジトリ画面を開きます。
2. `Settings` を開きます。
3. 左メニューの `Pages` を開きます。
4. `Build and deployment` の `Source` を `Deploy from a branch` にします。
5. `Branch` を `main` にします。
6. `Folder` を `/root` にします。
7. `Save` を押します。
8. 数十秒から数分待つと、Pagesの公開URLが表示されます。

### 4. 会社PCで使う

1. GitHub Pagesの公開URLを会社PCのブラウザで開きます。
2. 基本は「スライド編集」で、各スライドのフレーム写真追加から会社PC内の写真を選択します。必要な場合だけ「写真ライブラリ」に候補写真をまとめて追加します。
3. 写真はブラウザ内で処理されます。
4. 写真はGitHub Pages側にもGitHubリポジトリにも送信されません。
5. 編集途中は「軽量JSON保存」または「画像込みバックアップ保存」でプロジェクトファイルを書き出します。
6. 後日再開するときは「保存済みプロジェクトを開く」からJSONを読み込みます。

### 5. 公開前チェック

- `index.html` がリポジトリのルートにある
- `css/style.css` を相対パスで読み込んでいる
- `js/app.js` などを相対パスで読み込んでいる
- ローカルPC固有の絶対パスが含まれていない
- 外部CDNに依存していない
- 写真をGitHubにアップロードしていない
- 写真読み込みは `input type="file"` またはドラッグ＆ドロップで行う
- OpenAI APIキーや外部APIキーを使っていない

## 既知の制限

- 100枚程度の写真を想定しています。写真の枚数やPC性能によって読み込みに時間がかかります。
- 軽量JSON保存は画像データを含まないため、読み込み後に写真そのものは表示されません。
- 画像込みバックアップ保存は画像データを含むため、写真が多い場合は保存ファイルが大きくなります。
- 画像データを含めた保存もlocalStorageではなくIndexedDBとJSONファイルを使います。
- HEICは対応していません。JPG変換してから利用してください。
- ブラウザやプリンター設定によってPDFの余白や縮尺が変わることがあります。
- PPTXは見た目一致を優先した画像貼り付け方式です。PowerPoint上で文字や写真を個別編集する用途には向きません。
- 写真の顔認識や内容解析は行いません。写真メモはユーザーが入力してください。
